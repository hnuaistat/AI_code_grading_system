## 🔄 협업 패턴

### 패턴 1: 순차 협업 (Sequential)
```
사용자 요청
    ↓
DevOps (구조 검토)
    ↓
백엔드 (API 설계)
    ↓
노트북 엔지니어 (파일 처리)
    ↓
AI 채점 엔지니어 (채점 로직)
    ↓
인증 엔지니어 (보안 추가)
    ↓
프론트엔드 (UI 구현)
    ↓
완료 ✅
```

### 패턴 2: 병렬 협업 (Parallel)
```
                사용자 요청
                    ↓
        ┌───────────┼───────────┐
        ↓           ↓           ↓
    백엔드      노트북      프론트엔드
    (API 설계) (파싱)     (UI 설계)
        ↓           ↓           ↓
        └───────────┼───────────┘
                    ↓
        (완료 후 통합)
                    ↓
                완료 ✅
```

### 패턴 3: 피드백 루프 (Iterative)
```
설계 → 구현 → 테스트 → 피드백 → 개선 → 반복
←─────────────────────────────────────→
```

## 🎯 작업 요청 체계

### 1단계: 요청 작성
```
에이전트 선택 → 구체적 내용 → 파일 명시 → 예상 결과
```

### 2단계: 에이전트 검토
```
요청 분석 → 범위 확인 → 다른 에이전트 필요 여부 판단
```

### 3단계: 구현
```
자신의 영역에서만 작업 → 필요시 다른 에이전트와 조율
```

### 4단계: 완료
```
테스트 → 문서 작성 → 결과 보고
```
